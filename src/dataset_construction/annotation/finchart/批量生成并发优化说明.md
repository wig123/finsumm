# 批量生成并发优化说明

## 📋 修改文件
- `$HOME/Documents/githublearning/annotation-system/public/manage.js`

## 🎯 优化目标
将批量生成功能从**串行执行**改为**并发执行**，提升处理速度。

---

## 🔄 修改内容

### 原逻辑（串行）
```javascript
// 逐个处理，每个请求间隔3秒
for (const imageId of selectedImages) {
  await api.ai.generate(image.path, prompt);
  await api.annotations.create(...);
  await sleep(3000); // 等待3秒
}
```

**问题**：
- ❌ 100张图片需要 100 × 3秒 = 5分钟+
- ❌ 资源利用率低
- ❌ 用户等待时间长

---

### 新逻辑（并发控制）
```javascript
// 创建5个并发工作线程，共享任务队列
async function batchGenerateWithConcurrency(imageIds, prompt, concurrency = 5) {
  const queue = [...imageIds];
  
  // 工作线程：从队列取任务并处理
  async function worker(workerId) {
    while (queue.length > 0) {
      const imageId = queue.shift();
      // 处理图片生成...
    }
  }
  
  // 启动5个工作线程并行执行
  const workers = Array(5).fill(null).map((_, i) => worker(i + 1));
  await Promise.all(workers);
}
```

**优势**：
- ✅ 100张图片理论耗时 ≈ 100 ÷ 5 = 20次请求周期
- ✅ 速度提升 **5倍**
- ✅ 避免 429 限流（5个并发在大多数API限制内）
- ✅ 失败自动重试（429错误等待5秒后重试1次）

## 🔍 故障排查

### 问题1：大量429错误
**原因**：并发数超过API限制  
**解决**：降低并发数（改为3）

### 问题2：进度卡住不动
**原因**：某个请求超时  
**解决**：查看控制台日志，找到卡住的 Worker ID

### 问题3：失败率高
**原因**：网络不稳定或API问题  
**解决**：
1. 检查网络连接
2. 查看 `.env` 文件中的 API 配置
3. 查看控制台失败详情

## 📝 技术细节

### 并发控制原理
使用 **工作池模式**（Worker Pool Pattern）：

1. **任务队列**：所有图片ID存入队列
2. **工作线程**：创建N个异步工作函数
3. **任务分配**：每个工作线程从队列取任务（`queue.shift()`）
4. **并行执行**：`Promise.all()` 等待所有工作线程完成
5. **队列为空**：所有工作线程自动退出

### 为什么不用 Promise.all() 直接并发？
```javascript
// ❌ 不推荐：无法控制并发数
await Promise.all(imageIds.map(id => process(id)));
// 如果有1000张图片，会同时发起1000个请求！
```

```javascript
// ✅ 推荐：固定并发数
const workers = Array(5).fill(null).map(() => worker());
await Promise.all(workers);
// 始终只有5个请求并发
```
