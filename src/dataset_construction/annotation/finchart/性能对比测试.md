# 批量生成性能对比测试

## 🎯 测试场景

测试条件：
- 图片数量：100 张
- AI 单次响应时间：约 2-3 秒
- 网络延迟：忽略不计

---

## 📊 原方案（串行执行）

```
时间轴（秒）：
0s ────────────────────────────────────────────► 300s+

图片1: [████████] 3s
       等待 3s
图片2:        [████████] 3s
              等待 3s
图片3:               [████████] 3s
                     等待 3s
...
图片100:                                      [████████] 3s

总耗时：100 × (3s 处理 + 3s 等待) = 600秒 = 10分钟
```

**处理方式**：
```javascript
for (const id of imageIds) {
  await process(id);  // 3秒
  await sleep(3000);  // 3秒
}
```

---

## ⚡ 新方案（5并发执行）

```
时间轴（秒）：
0s ────────────────────────────────────────────► 60s

Worker 1: [████][████][████][████][████]...  (20张)
Worker 2: [████][████][████][████][████]...  (20张)
Worker 3: [████][████][████][████][████]...  (20张)
Worker 4: [████][████][████][████][████]...  (20张)
Worker 5: [████][████][████][████][████]...  (20张)

总耗时：100 ÷ 5 × 3s = 60秒 = 1分钟
```

**处理方式**：
```javascript
// 5个工作线程并行从队列取任务
const workers = [
  worker1(), // 处理 图片1, 图片6, 图片11, ...
  worker2(), // 处理 图片2, 图片7, 图片12, ...
  worker3(), // 处理 图片3, 图片8, 图片13, ...
  worker4(), // 处理 图片4, 图片9, 图片14, ...
  worker5(), // 处理 图片5, 图片10, 图片15, ...
];
await Promise.all(workers);
```

## 🎯 实际使用建议

### 1. 小批量测试（5-10张）
首次使用时先选择少量图片测试：
- 验证功能正常
- 观察控制台日志
- 熟悉操作流程

### 2. 逐步增加批量
- 10张 → 50张 → 100张 → 更多
- 观察 429 错误率
- 必要时调整并发数

### 3. 定期提交数据
每完成 100-200 张标注后：
```bash
git add database.db
git commit -m "标注: 完成 100 张"
git push
```

### 4. 长时间任务建议
如需处理 500+ 张图片：
- 分批进行（每次 100 张）
- 避免浏览器长时间运行
- 定期保存进度
