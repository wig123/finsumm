# 时间序列摘要生成方法

## 概述

通用的金融时间序列自动摘要系统,输入 (日期, 数值) 数据,输出结构化Markdown摘要 + 可视化图表。

**核心特点**:
- 仅需两列数据,无需收益率/成交量等衍生字段
- 6个分析维度,全面覆盖趋势/极值/波动/风险
- 自适应参数,支持半年到10年的数据范围
- LLM友好的Markdown输出格式

---

## 分析维度

### 1. 全局统计
基础统计量: 均值、标准差、偏度、峰度、年化波动率

### 2. 分段分析 (pwlf)
- 方法: 分段线性拟合
- 参数: n_segments=12, R²≈0.93
- 输出: 12个线性段,标注上升/下降/横盘

### 3. 变点检测
- 基于分段边界的突变强度
- 类型: 趋势反转/斜率变化/微小变动
- 强度: 1-5星评级

### 4. 极值点检测 (scipy_peaks)
- 参数: prominence=1.0×std, distance=3%
- 输出: 峰点和谷点,按显著性排序

### 5. 波动体制识别 (自适应)
- 平滑窗口: 数据长度的3% (最小10天,最大60天)
- 最短体制: 数据长度的2.5% (最小5天,最大60天)
- 分位数: 25%/75% (低/中/高波动)

### 6. 风险度量
- 最大回撤 (时间区间 + 幅度)
- 最长连跌天数

---

## 核心算法

### pwlf 分段线性拟合
```python
# 将时间序列拟合为12个线性段
my_pwlf = pwlf.PiecewiseLinFit(x, y)
breaks = my_pwlf.fit(12)
slopes = my_pwlf.slopes  # 每段斜率
```

**优势**:
- 高拟合度 (R²>0.93)
- 可解释性强 (每段有明确趋势)
- 自动识别转折点

### scipy.signal.find_peaks 极值检测
```python
# prominence: 峰的显著性阈值 (1.0×标准差)
# distance: 峰之间最小距离 (数据的3%)
peaks, props = find_peaks(y, prominence=std, distance=0.03*n)
```

**优势**:
- 过滤噪声,只保留显著极值
- 自动排序,突出重要峰谷

### 自适应波动体制识别
```python
# 平滑窗口 = 数据长度的3%
smooth_window = max(10, min(60, int(n * 0.03)))

# 最短体制 = 数据长度的2.5%
min_duration = max(5, min(60, int(n * 0.025)))

# 极端分位数 (识别主要体制)
low_mid = 25th percentile
mid_high = 75th percentile
```

**自适应效果**:
| 数据长度 | 平滑窗口 | 最短体制 |
|---------|---------|---------|
| 半年(126天) | 10天 | 5天 |
| 1年(252天) | 10天 | 5天 |
| 5年(1260天) | 36天 | 30天 |
| 10年(2520天) | 60天 | 60天 |

## 使用示例

```python
from pathlib import Path
import pandas as pd

# 加载数据 (仅需 DATE 和 VALUE 两列)
df = pd.read_csv('data.csv', parse_dates=['DATE'], index_col='DATE')

# 初始化分析器
analyzer = TimeSeriesAnalyzer(df, series_name='Stock_A')

# 执行分析
results = analyzer.analyze_all()

# 生成可视化
visualizer = Visualizer(df, results, output_dir='output')
visualizer.create_all()

# 生成Markdown摘要
md_gen = MarkdownGenerator(results, output_dir='output')
md_gen.generate_all()
```

**执行时间**: ~100秒 (1241个数据点)

## 参数配置

### 核心参数 (已调优)

| 模块 | 参数 | 默认值 | 说明 |
|------|------|--------|------|
| pwlf | n_segments | 12 | 分段数量 |
| scipy_peaks | prominence | 1.0×std | 峰显著性 |
| scipy_peaks | distance | 3% | 峰最小间隔 |
| volatility | smooth_pct | 3% | 平滑窗口比例 |
| volatility | min_duration_pct | 2.5% | 最短体制比例 |
| volatility | quantiles | 25%/75% | 分位数阈值 |

### 何时需要调整

**高频数据** (分钟/小时):
- pwlf: n_segments → 24-48
- volatility: smooth_pct → 1-2%

**极端波动市场**:
- scipy_peaks: prominence → 1.5×std
- volatility: quantiles → 20%/80%

## 设计理念

### 1. 通用性优先
- 仅需 (日期,数值),不依赖特定领域字段
- 适用于股票/期货/外汇/商品等所有金融时间序列

### 2. 自适应参数
- 根据数据长度动态调整窗口/阈值
- 避免固定参数在不同时间跨度下失效

### 3. LLM友好
- Markdown结构化输出
- 表格化呈现,便于LLM解析
- 清晰的导航链接

### 4. 可解释性
- 所有指标都有金融含义
- 避免黑盒模型 (如深度学习)
- 便于人工验证和调试

## 局限性

1. **仅支持单变量**: 无法处理价格+成交量的联合分析
2. **无因果推断**: 不解释体制切换的原因
3. **静态分析**: 不支持流式/增量更新
4. **日度数据优化**: 高频数据需调整参数
