# 训练配置教训（V4 踩坑记录）

**日期**：2026-03-30

## 必须设置的 LlamaFactory DPO 配置

```yaml
# 这两项缺一不可，否则训练结果会严重劣化
create_new_adapter: true      # DPO 创建独立 adapter，不覆盖 SFT adapter
template: qwen3_vl_nothink    # 禁用 thinking 模式
```

## 踩坑详情

### 1. `create_new_adapter: true` 缺失

**现象**：rewards/chosen ~57（异常高），loss ~2.3，评估 0.573（灾难性）
**原因**：不设此项时 LlamaFactory 直接修改 SFT adapter 权重。推理时先 merge SFT 再 merge DPO，SFT 被 apply 两次。
**修复后**：rewards/chosen ~0.9（正常），loss ~0.6，评估 0.738

### 2. `template: qwen3_vl` vs `qwen3_vl_nothink`

**差异**：`qwen3_vl` 使用 `ReasoningTemplate`（生成 `<think>...</think>`），`qwen3_vl_nothink` 是普通模板
**影响**：DPO 训练数据中的 chosen/rejected 不含 thinking tokens，用 thinking 模板会导致格式错配
**规则**：LF-2 baseline 用的是 `qwen3_vl_nothink`，所有实验必须统一

## 推理双层 LoRA 加载（正确方式）

```python
# 1. 加载 base model
model = Qwen3VLForConditionalGeneration.from_pretrained(base_path)
# 2. 加载 SFT adapter → merge 进 base
model = PeftModel.from_pretrained(model, sft_adapter_path)
model = model.merge_and_unload()
# 3. 加载 DPO adapter → merge（因为 create_new_adapter=true，这是独立增量）
model = PeftModel.from_pretrained(model, dpo_adapter_path)
model = model.merge_and_unload()
```
