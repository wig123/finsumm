## 一、V4 目标

基于 V2/V3 FCPO 实验的教训，探索**能够真正利用源数据可验证性**的方法。

### 已确认的事实（不可忽视）

| 事实 | 数据支撑 |
|------|---------|
| 标准 DPO (β=0.1, rank=256) 略超 SFT | LF-2=0.755 vs SFT=0.744 (+1.5%) |
| FCPO per-sample margin 无效 | Shuffle ≈ FCPO，LF-FCPO 全部低于 SFT |
| r_fact v2 (3-flash 1-100) 信号好于 v1 | v2: 56% positive, 21% tie vs v1: 39% positive, 42% tie（仅 Tier1/2） |
| r_fact v1/v2 方向一致性仅 47% | LLM fact 评分自身噪声大 |
| T0.6 扩增数据反而拉低分数 | merged_2815=0.745 < 原始 1700=0.755，**待排查原因** |
| ms-swift 推理不可靠 | mode collapse, 0.630 vs LF 0.755 |
| LlamaFactory 推理可靠 | 分数分布合理，与 SFT 一致 |

## 二、候选方案（按优先级排序）

### 方案 A：Best-of-N + Fact Reranking（推理时利用 fact 信号）

**核心思路**：不改训练，在推理时用 r_fact 做 reranking。

```
SFT/DPO 模型 → 生成 N 个候选 → r_fact + r_quality 评分 → 选最好的
```

**为什么可能 work**：
- r_fact 在单个样本内的绝对值可能有区分度（chosen 内部的多个候选）
- 训练时 Δr_fact 小是因为 chosen/rejected 都来自同一 SFT 模型，差异有限
- 推理时 N 个候选的多样性更大，fact 信号的区分度可能更好

**实验设计**：
1. 用 SFT 模型 + temperature=0.7 生成 N=8 个候选
2. 用 r_fact（规则提取 + raw.csv 验证）评分
3. 用 r_quality（Judge 评分）评分
4. 组合 reranking: `score = α × r_fact + (1-α) × r_quality`
5. 对比：Best-of-1 (greedy) vs Best-of-N (random) vs Best-of-N (r_fact) vs Best-of-N (r_quality) vs Best-of-N (combined)

**优势**：零训练成本，直接验证 fact 信号的价值
**风险**：推理成本 N 倍

### 方案 B：Fact-Guided Data Selection（用 fact 信号筛数据）

**核心思路**：不改 loss，用 r_fact 筛选高质量偏好对做标准 DPO。

```
1700 偏好对 → r_fact 过滤 → 高质量子集 → 标准 DPO 训练
```

**策略选项**：
- B1: 去除 Δr_fact < -0.1 的样本（chosen 事实更差的不该作为正例）
- B2: 只保留 Δr_fact > 0.1 的样本（高信号子集）
- B3: 按 r_fact_chosen 排序，只保留 top-K（确保 chosen 本身质量高）

**为什么可能 work**：
- V3 实验中 clean-B-fixed (846 条清洗后数据) = 0.746，接近 1700 条的 0.755
- 说明去掉"坏数据"没有太大损失，可能有剩余空间

**风险**：数据量减少，可能不如全量 DPO

### 方案 C：DPO 数据扩增 + 质量控制

**核心思路**：提升 DPO 本身的效果，不依赖 fact 信号。

```
当前 1700 对 → 扩增到 3000-5000 对 → 标准 DPO
```

**方法**：
- C1: 用 SFT 剩余图表（SFT 5099 条中未用于 DPO 的 3399 条）生成新偏好对
- C2: 多模型候选（用不同 checkpoint 生成候选，增加多样性）
- C3: 质量分级 DPO（按 Judge 分差分组，先学大差异再学小差异）

### 方案 D：SimPO / 其他 DPO 变体

**核心思路**：在标准 DPO 基础上换更好的 loss。

| 变体 | 已测试 | 结果 |
|------|--------|------|
| Standard DPO (sigmoid) | ✅ | LF-2=0.755 (best) |
| SimPO | ✅ | h20_simpo=0.716 (差) |
| β=0.05 | ✅ | 0.726-0.741 |
| β=0.2 | ✅ | 0.752 |
| 2 epoch | ✅ | 有过拟合风险 |

SimPO 已测试过，效果不好。其他未测试的变体：
- D1: IPO loss
- D2: KTO (不需要偏好对)
- D3: ORPO
- D4: DPO + SFT 辅助损失 (rpo_alpha)

### 方案 E：放弃 FCPO 叙事，转向 DPO 数据质量研究

**核心思路**：论文叙事从"改 loss"转向"改数据"。

> "在金融图表结构化总结领域，DPO 数据的质量和来源比偏好优化算法的选择更重要"

需要的消融实验：
- 数据量消融（500/1000/1700/3000）
- 数据来源消融（合成 vs 真实）
- 偏好对构建方法消融（Judge 评分 vs Fact-guided vs 随机）
- Judge 模型消融（不同 Judge 构建的偏好对）

## 三、推荐执行顺序

```
Phase 1: 方案 A (Best-of-N)     ← 零训练成本，快速验证 fact 信号价值
         方案 B (Data Selection)  ← 低成本，1-2 个训练实验
         可并行

Phase 2: 根据 Phase 1 结果决定
         如果 A 有效 → 深入 reranking + 训练结合
         如果 B 有效 → 方案 C 扩增数据
         如果都无效 → 方案 E 转叙事
```

## 四、评估标准（V4 升级）

### 必须做的改进

| 问题 | V2/V3 | V4 要求 |
|------|-------|---------|
| 样本量 | L2 (200) 为主 | 关键实验必须跑 **L3 (1000)** |
| 统计检验 | 未做 | **3 seed + paired bootstrap** |
| 推理框架 | ms-swift + LF 混用 | **统一 LlamaFactory** |
| 交叉 Judge | 未做 | Top-3 实验用 **GPT-5 交叉验证** |

### 评估流程

```
L1 (训练指标) → 淘汰明显失败
L2 (200 样本) → 筛选 top-K
L3 (1000 样本) × 3 seeds → 统计检验，写入论文
L4 (人工 50-100) → 最终验证
```
